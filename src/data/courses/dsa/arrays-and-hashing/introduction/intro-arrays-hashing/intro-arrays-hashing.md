## Arrays & Hashing — The Foundation of DSA

Arrays and hash maps are the **most fundamental data structures** in programming and the **most frequently tested** topic in FAANG interviews. Nearly every coding interview begins with array/hashing problems, making this the essential starting point.

<br>

## Why Arrays & Hashing First?

- **Most asked category** — ~30% of all FAANG interview questions involve arrays or hash maps
- **Foundation for everything** — Two pointers, sliding window, sorting, and prefix sum all build on arrays
- **O(1) lookup** — Hash maps transform O(n²) brute-force solutions into O(n) optimal ones
- **Simple to code** — Focus on problem-solving logic, not complex data structure implementation

<br>

## Core Patterns You'll Learn

1. **Hash Map Lookup** — Use hash maps to find complements, check existence, count frequencies (Two Sum, Contains Duplicate, Valid Anagram)
2. **Frequency Counting** — Count occurrences to group, validate, or find patterns (Group Anagrams, Majority Element, Valid Sudoku)
3. **In-Place Array Manipulation** — Modify arrays without extra space using index tricks (Set Matrix Zeroes, Rotate Image, Next Permutation)
4. **Sequence Detection** — Use hash sets for O(n) sequence finding (Longest Consecutive Sequence, First Missing Positive)

<br>

## Key Techniques

| Technique | When to Use | Time | Space |
|-----------|-------------|------|-------|
| Hash Map | Need O(1) lookup/insert | O(n) | O(n) |
| Hash Set | Need O(1) existence check | O(n) | O(n) |
| Sorting + Scan | Need ordered comparison | O(n log n) | O(1) |
| Index as Hash | Values in range [1, n] | O(n) | O(1) |
| Prefix Product | Product without division | O(n) | O(n) → O(1) |

<br>

## FAANG Interview Frequency

- **Google:** Two Sum, Product of Array Except Self, First Missing Positive
- **Meta:** Group Anagrams, Valid Sudoku, Longest Consecutive Sequence
- **Amazon:** Contains Duplicate, Encode/Decode Strings, Rotate Image
